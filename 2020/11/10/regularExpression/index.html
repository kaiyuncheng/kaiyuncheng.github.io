[JS] 簡單認識正規表達式 Regular Expression

前言

這篇會來介紹正規表達式，第一次看到正規表達式時，只會想說這是什麼鬼…為何一小串就可以做完一般判斷式可能要寫好幾行的事。如果可以更深入了解活用的確可以在coding中幫助極大。因此這次我會藉由這篇讓我自己更釐清觀念。

介紹

正規表達式(Regular Expression, RegEx)是被用來匹配字串中字元組合的模式。
常搭配與這些字串方法(match、replace、search、split)來使用。

為什麼使用正則表達式？

> 用來測試字符串是否符合模式

可以用於驗證，像是電話,Email,信用卡號碼是否輸入正確

> 收尋匹配的字串

> 取代匹配到的字串

> 擷取出匹配到的字串

正則表達式使用規則

簡易字元

例如： /abc/

  • .replace(/abc/,’a’); 用’a’來取代匹配到的’abc’
  • strB 因為是abC的C為大寫，在/abc/後多加i則可以不管大小寫辨識到
  • strC 為ab c中有空白字元所以匹配不到
  • strD 在/abc/後多加g則可收尋到所有abc並且替換成a

特殊字元

例如： /ab*c/

  • *代表b可以是零或多個，所以abbbbbbc或是ac都可以匹配到

字元 解說
\ 反斜線為避開特殊字元，若要表現\則須用兩個反斜線\\
/^ 正規表示法的開始 /^A/則匹配A開頭的字元，例如Ant
$/ 正規表示法的結束 /A$/則匹配A結束的字元，例如nbA
所有正規表示法規則都必須放到這兩者之間。

$/ | 正規表示法的結束 /A$/則匹配A結束的字元，例如nbA

正規表示法的開始：/^
正規表示法的結束：$/
所有正規表示法規則都必須放到這兩者之間。
[A-Z] 表示英文大寫的文字格式，意味只可填寫 A 到 Z 之間的大寫字母。
\d 或 [0-9] 表示數字格式，意味只可填寫 0 - 9 的數字。
{9} 表示後面可有幾位數字。

舉例：[A-Z]{3}
以上表示要填三個大寫字母，並且只能填大寫字母 A 到 Z 之間，例如：ACE、AEZ、EDO……等，若後面不寫預設為 1 個。

.match(/abc/)會回傳一個符合包含abc的陣列

正規表達模式由數個簡易字元組成，例如 /abc/，或是由簡易字元及特殊符號組合而成，例如 /abc/、/Chapter (\d+).\d/ )。最後一個範例用到了括號，這在正規表達式中用作記憶組，使用括號的匹配將會被留到後面使用

判斷中英文字串的長度
因為中文佔兩個字元,而英文佔1個字元,若只有用string.length去判斷,則是不準確的,可用下面的function來取得真正的長度.

function getlengthb(str){
return str.replace(/[^\x00-\xff]/g,”**”).length;
}
